# Лабораторная работа №1: Duck Typing и контрактное программирование

## Цель
Освоить duck typing и контрактное программирование на примере источников задач.

## Задача
Реализовать подсистему, которая может принимать задачи из различных источников (файл, генератор, API-заглушка) через единый поведенческий контракт.

## Требования
- Все источники реализуют контракт `TaskSource` (метод `get_tasks()`).
- Контракт описан с помощью `typing.Protocol` и `@runtime_checkable`.
- Выполняется runtime-проверка соблюдения контракта.
- Архитектура позволяет добавлять новые источники без изменения существующего кода.

## Реализация

### Компоненты
- **Модель задачи** (`src/models/task.py`) – класс `Task` с полями `id` и `payload`.
- **Контракт** (`src/protocols/source_protocol.py`) – протокол `TaskSource`.
- **Источники задач**:
  - `FileSource` – читает задачи из текстового файла.
  - `Generate` – генерирует случайные задачи из списка повседневных дел.
  - `ApiSource` – возвращает фиксированный набор задач (имитация API).
- **Обработчик** (`src/processors/task_processor.py`) – проверяет контракт и выводит задачи.

### Запуск проекта
```bash
python3 main.py
```

### Запуск тестов
```bash
python3 -m pytest tests/
```
